## >>程式目的

`陳葳芃_交通大學_技術測驗(一).zip` 包含工研院服科中心實習考題(一)，目的在建立一個傷口癒合狀態自動評估工具，透過 `jupyternote book` 來撰寫 `python`。

---

## >>運行方式

解壓縮後直接使用 terminal `陳葳芃_交通大學_技術測驗(一).py` 即可
本 code 運行邏輯依據不同使用者分為 2 個區塊：

* **(1) 測試程式內定義的資料**
    由想了解本程式運作情形的其他開發者可直接選擇此模式，查看不同傷口癒合狀態下，本程式會給出不同對應等級的照護小叮嚀。

* **(2) Console 輸入**
    面向真實使用者進行個別化傷口癒合程度輸入。

---

## >>Assessment

除了基本要求外，我做了以下的建置：

* **(1) 互動感增加**
    使用 `time` 及一些小小的互動延遲功能來提升使用感受。

* **(2) 額外的例外處理**
    由於並非 GUI 介面而是使用者自行輸入，我特別考量了以下情形：
    * 傷口面積不能為負數
    * 疼痛等級必須在 0 到 10 之間
    * 滲液量輸入必須為 `(None, Light, Moderate, Heavy)` 其中一類別
    
    來避免 code 運行出現問題。考量輸入數字的地方是不是數字，有沒有物輸成字串的情形？若有，程式即可重新運行一次，並提供相對應警語。

    此外，針對滲液量輸入，我也設計為不管輸入大小寫，最後會利用 `capitalize` 來進行檢核，增加使用者輸入的便利性，也檢核程式運作的穩定性！

* **(3) 使用者輸入的確認**
    在使用者輸入完後提供確認階段，來確保使用者正確輸入的自己正確的傷口情況！

---

## >>可精進方向

* **(1) UX體驗提升**
    目前只純粹利用 `python` 達成我們基礎的功能，但對真實使用者體驗來說可能仍舊不夠，後續精進方向即可為設計 GUI 介面（by `tkinter`...），一方面可以提升不同年齡層的操作上手程度，二來可以透過圖形化的 buttom 避免使用者異常輸入導致噴錯的情形產生。

* **(2) 多語介面的建置**
    考量到需要此系統來輔助照護的客群可能為年長者，而多數年長者可能不具備操作此系統的能力，使用者能即為照顧年長者看護。因此，若能提升語言版本的切換，可能進而使醫院在與看護進行講解時，能降低溝通成本，讓醫護回歸診療本業，投入心力在真正的醫療診護上。

* **(3) 即時資訊提供**
    正式上線此程式，即可配對病患的個人資料，如果能放入醫囑、就診醫院資料，讓使用者能正確照護以及提供即時的就診資訊，因能使照護系統更加完善。
